💡 주요 설계 결정
================================================================================

1. 규칙 기반 감성분석 선택
   - 이유: 토큰 비용 절약, 빠른 응답, 한국어 주식 댓글 최적화
   - 장점: 비용 효율적, 실시간 처리 가능
   - 단점: 정확도 낮음 (딥러닝 대비)
   - 확장: transformers (KoBERT) 나중에 추가 가능

2. 동적 HTML 렌더링 (vs React/Vue)
   - 이유: 프로토타입 속도, 간단한 상호작용
   - 장점: 번들 크기 작음, SEO 친화적, 배포 간단
   - 단점: 상태 관리 제한
   - 확장: React로 마이그레이션 가능

3. 목업 데이터 사용 (vs DB)
   - 이유: 초기 개발 속도, DB 의존성 제거
   - 장점: 빠른 프로토타입, 테스트 용이
   - 단점: 실제 데이터 없음
   - 확장: PostgreSQL 연결 후 쿼리 변경

4. CSS 변수 + 다크 테마
   - 이유: 금융 애플리케이션에 적합한 전문성
   - 장점: 현대적, 접근성 좋음, 유지보수 용이
   - 확장: 라이트 테마 추가 가능
